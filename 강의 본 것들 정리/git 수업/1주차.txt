1주차 수업

- 버전 관리 : 프로젝트 상태 정보를 안다는 것

- git 초기화(initialize, init) : 기존 프로젝트를 git 프로젝트로 만들어 주는 것

- add/staging : commit에 반영할 파일을 선택하는 것

- commit : 현재 프로젝트의 상태를 저장 (누가, 언제, 변경내용, 커밋 메시지 등)

- commit 내역 : 이때까지 해온 커밋들을 시간 순서대로 무엇이 변경되었는지를 볼 수 있다.

- push : 프로젝트의 커밋 내역을 원격 리포에 올린다

- pull : 원격 리포에 있는 프로젝트 커밋들을 로컬 리포로 받아온다.

- clone : 프로젝트 초기에 원격 리포에 있는 프로젝트를 그대로 로컬 리포로 받아오는 것.

- tracking : 로컬 리포와 원격 리포를 연결시키는 것.
